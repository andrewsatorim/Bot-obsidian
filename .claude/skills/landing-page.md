# Landing Page Design

## Page Structure (Top to Bottom)

### 1. Hero Section
```
[Logo]  [Nav: Features | Pricing | Docs]  [CTA: Sign In]

# Main headline (benefit-driven, 5-10 words)
## Subheadline (how you deliver the benefit, 15-25 words)

[Primary CTA button]    [Secondary CTA: "See demo"]

[Hero image / product screenshot / short video]
```

**Rules:**
- Headline answers: "What do you get?"
- Subheadline answers: "How does it work?"
- CTA answers: "What do I do next?"
- Above the fold — no scrolling needed to understand the value
- One hero image showing the product in action

### 2. Social Proof Bar
```
Trusted by 5,000+ teams  |  [Logo] [Logo] [Logo] [Logo] [Logo]
```
- Company logos (grayscale, uniform size)
- Or metrics: "50K+ users", "99.9% uptime", "4.8★ rating"
- Place immediately after hero to build trust

### 3. Problem Section
```
## [Name the pain they feel]

Three columns describing specific pain points:
[Icon] Pain 1: description
[Icon] Pain 2: description  
[Icon] Pain 3: description
```

### 4. Solution / Features
```
## How [Product] works

Step 1: [Action] → screenshot
Step 2: [Action] → screenshot
Step 3: [Result] → screenshot
```
- 3 steps maximum
- Show, don't tell — use screenshots or animations
- Lead with the outcome, not the feature

### 5. Feature Grid
```
## Everything you need to [outcome]

[3x2 or 2x3 grid of features]
Each: Icon + Title (3 words) + Description (1 sentence)
```
- 6 features max
- Benefit-oriented titles: "Save hours" not "Batch processing"
- Consistent icon style

### 6. Testimonials
```
"Quote about specific result achieved."
— Name, Title, Company  [Photo]
```
- Include specific outcomes: "Reduced deploy time by 60%"
- Real photos, not stock
- 2-3 testimonials, not more
- Highlight key phrase in bold

### 7. Pricing
```
## Simple, transparent pricing

[Free]          [Pro]              [Enterprise]
$0/mo           $29/mo             Custom
3 projects      Unlimited          Everything in Pro +
Basic features  All features       SSO, audit logs
Community       Priority support   Dedicated support
[Start free]    [Start trial]      [Contact sales]
```
- 3 tiers maximum
- Highlight recommended tier visually
- Annual/monthly toggle
- Feature comparison below if needed

### 8. FAQ
```
## Frequently asked questions

[Accordion: 5-7 questions]
```
- Answer objections, not just questions
- "Can I cancel anytime?" → Yes, with reassurance
- "Is my data secure?" → Yes, with specifics

### 9. Final CTA
```
## Ready to [achieve outcome]?

[Same primary CTA as hero]
```
- Mirror the hero CTA
- Last chance to convert before footer

### 10. Footer
```
[Logo]
Product: Features, Pricing, Changelog, Roadmap
Resources: Docs, Blog, Community, Status
Company: About, Careers, Contact, Press
Legal: Privacy, Terms
© 2026 Brand. All rights reserved.
```

## Conversion Optimization

### Above the Fold Checklist
- [ ] Clear value proposition (what + for whom)
- [ ] One primary CTA (visible, contrasting color)
- [ ] Product visual (screenshot, not abstract illustration)
- [ ] Trust signal (logos, metrics, or badge)

### Performance Rules
- [ ] Page loads in < 2s (LCP)
- [ ] No layout shift (CLS < 0.1)
- [ ] Images optimized (WebP, lazy-loaded below fold)
- [ ] Critical CSS inlined

### A/B Testing Priority
1. Headline — highest impact on bounce rate
2. CTA text and color — directly affects conversion
3. Hero image/video — affects engagement
4. Social proof placement — affects trust
5. Pricing presentation — affects revenue

## Anti-Patterns (Don't)
- Slider/carousel in hero — users don't scroll them
- "We're the best" without proof — sounds desperate
- Multiple competing CTAs — confuses the user
- Stock photos of handshakes — kills credibility
- Walls of text — nobody reads them
- Popup on first visit — 90% close rate
- Hidden pricing — breeds distrust
