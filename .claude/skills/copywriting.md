# Copywriting

## Core Formulas

### AIDA (Attention → Interest → Desire → Action)
1. **Attention** — bold claim or unexpected hook
2. **Interest** — expand with relevant details
3. **Desire** — show the transformation/benefit
4. **Action** — clear CTA

### PAS (Problem → Agitate → Solution)
1. **Problem** — name the pain the reader feels
2. **Agitate** — amplify the consequences of not solving it
3. **Solution** — present your product as the answer

### BAB (Before → After → Bridge)
1. **Before** — current painful state
2. **After** — desired ideal state
3. **Bridge** — your product connects the two

## Headlines

### Power Patterns
- **How to [achieve desired outcome] without [pain point]**
- **[Number] ways to [benefit] in [timeframe]**
- **Stop [bad thing]. Start [good thing].**
- **The [adjective] way to [desired outcome]**
- **[Do thing] like [aspirational reference]**
- **What [successful group] know about [topic] that you don't**
- **[Benefit] — in [surprisingly short time/effort]**

### Rules
- Lead with the benefit, not the feature
- Be specific: "Save 4 hours/week" > "Save time"
- One idea per headline
- Under 10 words for impact (max 15)
- Test multiple variants — you can't predict winners

## CTAs (Call to Action)

### High-converting patterns
| Weak | Strong |
|------|--------|
| Submit | Get my free guide |
| Sign up | Start building for free |
| Buy now | Unlock premium features |
| Learn more | See how it works |
| Click here | Try it — no credit card needed |
| Download | Get instant access |

### Rules
- Start with a verb
- Make it about **them**, not you
- Reduce friction: "No credit card required", "Free for 14 days"
- Create urgency only when genuine
- One primary CTA per page/screen

## Product Descriptions

### Structure
1. **One-line hook** — what it is + key benefit
2. **Problem statement** — who it's for + what they struggle with
3. **Solution** — how your product solves it (3 bullet points max)
4. **Social proof** — numbers, logos, testimonials
5. **CTA** — next step

### Example
```
Ship faster with AI code review.               ← hook
Tired of waiting days for PR feedback?          ← problem
CodeBot reviews your pull requests in seconds:  ← solution
  • Catches bugs before your team does
  • Suggests improvements with code examples
  • Learns your codebase conventions
Trusted by 2,000+ engineering teams.            ← proof
Start free →                                    ← CTA
```

## Microcopy (UI Text)

### Empty States
```
Bad:  "No data"
Good: "No projects yet. Create your first one to get started."
```

### Error Messages
```
Bad:  "Error 422: Validation failed"
Good: "That email is already registered. Try logging in instead?"
```

### Loading States
```
Bad:  "Loading..."
Good: "Setting up your workspace..."
```

### Success Messages
```
Bad:  "Operation completed successfully"
Good: "Your changes are live! ✓"
```

### Form Labels & Placeholders
- Label tells **what**: "Email address"
- Placeholder shows **format**: "you@company.com"
- Helper text explains **why**: "We'll send your receipt here"
- Never use placeholder as label

## Voice & Tone Scale

| Context | Tone | Example |
|---------|------|---------|
| Marketing page | Confident, aspirational | "Build the future of finance" |
| Onboarding | Warm, encouraging | "Great start! Let's set up your profile" |
| Documentation | Clear, neutral | "Enter your API key to authenticate" |
| Error state | Helpful, calm | "Something went wrong. Try refreshing the page" |
| Success | Celebratory, brief | "You're all set!" |
| Legal/security | Precise, trustworthy | "Your data is encrypted end-to-end" |

## Writing Rules

1. **Cut fluff** — remove "very", "really", "actually", "in order to", "that being said"
2. **Active voice** — "We built X" not "X was built by us"
3. **Short sentences** — 15-20 words max. Mix with fragments for rhythm.
4. **Concrete > abstract** — "Reply in 2 hours" not "Fast response times"
5. **You > we** — focus on the reader: "You'll save time" not "We save you time"
6. **One idea per paragraph** — wall of text = zero readers
7. **Read it aloud** — if it sounds awkward, rewrite it
